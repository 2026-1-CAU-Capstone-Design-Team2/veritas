# VERITAS 데모 시나리오 — 전체 리스트

> 기업 실무에서 바로 와닿고, 일반 청중도 한눈에 가치를 이해할 수 있는 데모 시나리오 모음.
> 한 흐름 안에서 **외부 조사(AutoSurvey) → 정합성 검증(Cross-check) → 근거 기반 채팅(RAG) → 초안 생성(Draft) → 실시간 문서 보조(Proactive)** 를 모두 보여주고,
> 전 구간에서 **환각이 없다**는 것을 증명하는 것이 목표다.

---

## 0. 이 데모로 증명하는 것

| 기능 | 데모에서 보여줄 것 | "환각 없음" 증명 포인트 |
|---|---|---|
| **AutoSurvey** | 비공학·구어체 프롬프트 한 줄 → 웹 수집 → `final.md` 보고서 | 모든 문장이 수집 출처(`doc_NNN`)에 연결됨 |
| **Cross-check** | 내부 파일 숫자 ↔ 외부 보도 숫자 불일치를 자동 탐지 | 알고리즘(수치 비교)만 사용, LLM 추측 없음 |
| **RAG 채팅** | 수집·로컬 문서 근거 위에서만 답변, 인용 클릭 시 원문 팝업 | 근거 없으면 답하지 않음 / 출처 표기 |
| **table_query** | 로컬 CSV·XLSX의 합계·집계를 **원본 전체를 읽어** 정확 계산 | 표 숫자를 지어내지 않음 |
| **Draft(초안)** | 양식·톤에 맞춘 문서 생성, Cross-check 불일치를 초안에 명시 | 지식팩(웹+로컬) 근거로만 작성 |
| **Proactive(실시간 보조)** | 작성 중 이어쓰기(ghost)·수정 제안, 사용자가 수락/거절 | 커서 주변 맥락만 사용(문서 전체 무단 인용 금지) |

> **로컬 우선**: 로컬 문서(`local_private`)는 OpenAI 가속이 켜져 있어도 외부로 전송되지 않는다. Cross-check·RAG·검증·로컬 처리는 항상 로컬 LLM으로 수행된다. (데모 중 강조 포인트)

## 1. 데모 공통 사전 준비

1. **워크스페이스 생성** — 주제 1개 = `runs/` 아래 폴더 1개.
2. **로컬 폴더 등록** — `설정 → 로컬 접근 폴더 설정`에서 내부 파일이 든 폴더 추가 → 자동 스캔·색인.
   - 지원: `.md .txt .pdf .docx .xlsx .csv` (파일당 ≤50MB, 폴더당 ≤300개)
3. **내부 파일 준비** — §5 "Cross-check 발화 보장 체크리스트"를 반드시 따른다.
4. **(옵션) OpenAI 가속** — 조사 속도를 높이려면 `설정 → 고급 → OpenAI API` 키 등록. 끄면 전부 로컬.
5. **(권장) 리허설 1회** — 본 행사 전에 동일 프롬프트로 AutoSurvey를 한 번 돌려:
   - 웹이 실제로 반환하는 외부 수치를 `final.md`/배치 요약에서 확인하고,
   - 내부 파일 숫자를 그 값의 **±5~15%**(같은 단위·같은 지표명)로 맞춰 **반드시 Cross-check가 터지게** 보정한다.
   - 수집 문서가 캐시되어 본 행사에서 빠르고 안정적으로 재현된다.

## 2. 메인 시나리오 (골든 패스) — 재무/IR팀: 분기 실적 발표 자료 사전 검증

> **한 줄 요약**: "발표/공시 전에, 내가 만든 실적 요약 숫자가 시장에 알려진 숫자와 맞는지 자동으로 점검하고, 그 위에서 보고서 초안까지 환각 없이 작성한다."
> 가장 안정적이고(문서의 공식 예시), 일반 청중이 즉시 가치를 이해하는 시나리오. **메인 데모로 권장.**

### 준비물 — 내부 파일 예시 `2024_4분기_실적요약_초안.md`

```markdown
# 2024년 4분기 실적 요약 (내부 초안)

본 자료는 IR 발표 전 내부 검토용 초안이다.
2024년 4분기 연결 기준 영업이익은 15.8조원을 기록했다.
같은 기간 매출은 73조원으로 집계됐다.
영업이익률은 약 21.6% 수준으로 추정된다.
```

> 위 숫자(15.8조 등)는 **예시**다. 리허설에서 웹이 반환하는 실제 영업이익 수치를 확인한 뒤,
> 내부 값을 그 수치의 1.2배 이내(같은 "영업이익"·같은 "조원")로 보정해 불일치가 확실히 잡히게 한다.

### Step 1 — 외부 조사 (비공학·구어체 프롬프트)

- 조사 모드에서 입력:
  > **"삼성전자 작년 4분기 잠정 실적이 어땠는지, 영업이익이랑 매출 위주로 정리해줘."**
- (안정성 강화 옵션) 신뢰 출처를 고정하려면 `site:` 제약 사용:
  > `삼성전자 2024년 4분기 잠정 실적 site:news.samsung.com`
- **기대 결과**: 진행률이 단계별(주제어→검색계획→수집→요약→재계획→최종보고서)로 흐르고, `final.md` 보고서가 생성됨.
- **증명**: 평범한 한 문장 요청으로 자율 조사가 굴러간다. 키워드 라우팅이 아니라 LLM이 계획을 세운다.

### Step 2 — 정합성 검증 (Cross-check)

- `검증` 페이지 → **"검증 시작"**.
- 4개 작업이 수행됨: ① 섹션별 근거 배치 ② 문서별 신뢰도 ③ 외부 출처 간 합의/충돌 ④ **Cross-check(내부↔외부)**.
- **기대 결과**: "Cross-check 결과" 카드에 불일치 건이 표시됨 —
  > 예: **내부 초안: 영업이익 15.8조원 ↔ 외부 보도: 16.4조원** (같은 지표·같은 단위·1.2배 이내)
- **증명**: LLM 추측이 아니라 **순수 수치 비교 알고리즘**으로 불일치를 잡는다. 로컬 문서 본문은 외부로 나가지 않는다.

### Step 3 — 근거 기반 채팅 (RAG, 환각 없음)

- 채팅(RAG 모드)에서:
  > "외부 자료 기준 4분기 영업이익 컨센서스는 얼마였어?"
  > "매출은 전분기 대비 어떻게 변했어?"
  > "(근거 없는 질문) 5분기 전망치도 알려줘" → **자료에 없으면 단정하지 않음**을 시연.
- 요약 보고서 화면에서 인용 `[doc_NNN]` 클릭 → **원문 문단 하이라이트 팝업**.
- **증명**: 수집 문서 근거 위에서만 답하고, 출처를 추적할 수 있다.

### Step 4 — 초안 생성 (Draft, Cross-check 주입)

- `초안 작성` → 양식(예: **경영진 보고용 실적 요약**) · 톤 · 목차 선택 → 생성.
- **기대 결과**: 웹+로컬 지식팩 근거로 초안 작성. 이때 `KnowledgePackBuilder`가 Cross-check 불일치를 **"Cross-check Notes"로 LLM 컨텍스트에 주입** → 초안이 불일치를 숨기지 않고 명시/주의 표기.
- **증명**: 생성 문서가 검증 결과와 일관된다(환각으로 한쪽 숫자를 임의 채택하지 않음).

### Step 5 — 실시간 문서 보조 ① 추가 작성 (Native 편집기 · 이어쓰기 ghost)

> VERITAS 자체 편집기에서는 **이어쓰기(`next_sentence`) ghost text**가 핵심. 커서 신뢰도(native_cursor ≈ 0.95)가 높아 안정적으로 뜬다.

- 자체 편집기에 초안을 열고, 한 문단을 쓰다가 **문장을 미완성으로 두고 잠깐 멈춤**(예: "이번 분기 실적의 핵심은 ").
- **기대 결과**: 디바운스(≈800ms) 후 회색 **ghost 텍스트**로 다음 문장 제안.
  - **TAB / 복사** = 수락, **ESC** = 거절, **"다시"** = 재생성.
- **거절 사다리 시연**(같은 위치 반복 거절 시 점점 다른 제안):
  | 거절 횟수 | 동작 | LLM이 받는 맥락 |
  |---|---|---|
  | 0 | 기본 ghost | 직전 prefix |
  | 1 | 표현/구조 바꿔 재시도 | `[직전 N문장]` + 거절 문장 회피 |
  | 2 | 방향·주제 자체를 다르게 | `[현재 문단 전체]` + 거절 문장 회피 |
  | 3 | 그 위치 180초 cooldown | — |
- **증명**: 커서 주변 맥락만으로 자연스럽게 이어 쓴다(문서 전체를 무단 인용하지 않음). 거절을 학습해 반복하지 않는다.

### Step 6 — 실시간 문서 보조 ② 수정 제안 (외부 앱 Word · 보조 카드)

> **수정 계열 제안**(단락 재작성·교정·흐름·분할)은 현재 **외부 앱 보조 카드 경로**에서 렌더된다. (native 편집기는 inline-diff/marker 렌더러 미구현 → 수정 제안은 카드로 표시) Word처럼 **UIA 캐럿**을 지원하는 앱을 쓰면 신뢰도(0.75~0.95)가 충분하다. OCR만 되는 앱은 신뢰도가 낮아 제안이 안 뜰 수 있다.

- 떠 있는 보조 창을 켠 상태로 **Word**에서 초안을 편집하며 상황을 만든다:
  | 만들 상황 | 뜨는 제안(task) | 카드 색 |
  |---|---|---|
  | 한 단어를 연달아 두 번 입력 / 200자 넘는 한 문장 | **local_copyedit**(최소 교정) | 빨강 |
  | 80자↑ 단락에서 붙여넣기·되돌리기·잦은 수정(churn) | **paragraph_rewrite**(단락 재작성) | 주황 |
  | 앞뒤 단락이 있는 상태로 잠깐 멈춤(문서 300자↑) | **logic_flow_review**(흐름 한 가지 수정) | 회색 |
  | 한 단락이 500자↑ 로 길어진 채 멈춤 | **long_paragraph_split**(2~3단락 분할) | 주황 |
  | 큰 삭제/붙여넣기 직후 그 자리에서 멈춤 | **recovery_or_integration_note**(복구) | 회색 |
  | 문단 끝/미완성 문장에서 2초+ 멈춤 | **next_sentence**(이어쓰기) | 파랑 |
- 카드는 **본문**(그대로 붙여넣을 텍스트)과 **설명:**(이유)으로 분리되어 표시됨.
- **증명**: 작성 맥락을 읽어 "언제·무엇을·어떻게" 제안할지 스스로 판단한다. 키워드 사전이 아니라 구조 신호로 트리거한다.

### (정리) 메인 시나리오가 한 번에 증명하는 것

조사 → 검증 → 채팅 → 초안 → 실시간 보조까지, **숫자·문장 모두 출처/맥락에 묶여** 환각이 끼어들 자리가 없다는 것.

## 3. 대안 시나리오 (부서별 — 같은 골든 패스에 주제만 교체)

각 시나리오는 §2의 5단계 구조를 그대로 쓰되 프롬프트/내부 파일/초안 양식만 바꾼다.

### B. 전략기획·마케팅: 경쟁사·시장 조사 vs 내부 추정치
- 프롬프트: **"경쟁사 LG전자 작년 연간 매출이랑 영업이익이 얼마였는지 조사해줘."**
- 내부 파일(`시장분석_보고서.docx`): *"경쟁사 연간 매출은 약 OO조원 수준으로 추정된다."*
- Cross-check: **우리가 추정한 경쟁사 숫자 ↔ 실제 발표치** 괴리.
- 초안: "경쟁사 동향 요약". / 강점: 거의 모든 기업의 일상 업무, 매우 직관적.

### C. 영업·사업개발: 제안서에 들어갈 시장 데이터 무환각 검증
- 프롬프트: **"OO 시장 규모가 작년에 얼마였고 성장률이 어떤지 조사해줘."**
- 내부 파일(`사업제안서_초안.docx`): 제안서에 인용한 시장 규모/성장률 숫자.
- Cross-check: **제안서 숫자 ↔ 외부 출처** → 고객 제출 문서에 틀린/지어낸 숫자 차단.
- 초안: "사업 제안서 - 시장 현황" 섹션(출처 인용 포함). / 강점: "환각 없는 글쓰기" 정체성을 가장 직접 시연.

### D. HR: 인건비·연봉 벤치마킹
- 프롬프트: **"올해 최저임금 시급이랑 작년 대비 인상률 조사해줘."**
- 내부 파일(`인건비_계획서.docx`): *"2025년 최저임금은 시급 9,920원으로 가정한다."*
- Cross-check: **가정 시급 9,920 ↔ 고시 10,030**(1.01배). / 강점: 숫자가 안정적이라 발화 신뢰도 높음.

### E. 구매·조달: 원자재·시장 가격 검증 (+ table_query)
- 프롬프트: **"작년 국제 금 평균 시세가 얼마였는지 조사해줘."**
- 내부 파일: 월별 구매단가 `CSV/XLSX` + 본문에 *"연평균 매입 단가는 OOO원이었다."*
- 채팅(table_query): **"상반기 평균 매입 단가는?"** → 요약이 아니라 **원본 파일 전체를 읽어 정확 계산**.
- Cross-check: 내부 가정 단가 ↔ 외부 시세. / 강점: **표 숫자 무환각**까지 추가로 증명.

## 4. 실시간 문서 보조(Proactive) 시연 레퍼런스

### 4.1 두 가지 표면(Surface)과 무엇이 뜨는지

| 표면 | 캡처 방식 | 신뢰도 | 실제 렌더되는 task |
|---|---|---|---|
| **Native 편집기** (VERITAS 자체) | 커서 직접 읽기(native_cursor) | ≈0.95 | **`next_sentence`(이어쓰기 ghost)** — TAB 수락. (수정 계열은 inline-diff/marker 렌더러 미구현으로 native에서는 생략됨) |
| **외부 앱** (Word 등) | UIA 캐럿 / OCR | UIA 0.75~0.95 · OCR 0.20~0.65 | 보조 **카드**로 모든 task(아래 표). OCR만 되는 앱은 신뢰도 미달로 안 뜰 수 있음 → **UIA 지원 앱(Word) 권장** |

### 4.2 Task 종류별 트리거 조건 (코드 기준) · 시연 방법

| Task | 트리거 조건(요지) | 시연 방법 | 카드색 |
|---|---|---|---|
| `next_sentence` | anchor 신뢰도 충분 + 문단 끝/미완성 문장 + 멈춤(native: 디바운스 / 외부: idle≥2s) | 문장을 미완성으로 두고 멈춤 | 파랑 |
| `local_copyedit` | 신뢰도≥0.60 + 반복 단어 또는 200자↑ 한 문장 | 같은 단어 연달아 2번 입력, 또는 아주 긴 한 문장 | 빨강 |
| `paragraph_rewrite` | 신뢰도≥0.60 + 단락 80자↑ + churn≥0.30 또는 undo/paste | 단락을 붙여넣기/되돌리기로 여러 번 고침 | 주황 |
| `logic_flow_review` | 신뢰도≥0.65 + 앞/뒤 단락 존재 + 멈춤 + 문서 300자↑(또는 단락 200자↑) | 단락 3개 이상 쓴 뒤 가운데서 멈춤 | 회색 |
| `long_paragraph_split` | 신뢰도 충분 + 단락 500자↑ + 멈춤 | 한 단락을 500자 넘게 길게 쓰고 멈춤 | 주황 |
| `recovery_or_integration_note` | 신뢰도 충분 + 최근 변경이 현재 위치와 겹침 + 큰 삭제/붙여넣기/undo | 큰 덩어리 삭제·붙여넣기 직후 그 자리에서 멈춤 | 회색 |
| `evidence_or_citation_prompt` | **자동 트리거 없음** (어휘 키워드 heuristic 영구 폐기). 명시적 호출 경로로만 | 데모에서는 다루지 않음(설계 원칙 설명용) | — |

### 4.3 피드백 동작과 거절 사다리 (native 이어쓰기)

- 수락: **TAB** 또는 복사 · 거절: **ESC** · 재생성: **"다시"** · 위치 어긋남: **wrong_anchor**.
- 같은 위치 반복 거절 → 0(기본) → 1(직전 N문장 맥락+표현 변경) → 2(문단 전체 맥락+방향 변경) → 3(180초 cooldown).
- 다른 위치로 이동하면 사다리 초기화. (수락이 사다리를 클리어)
- **증명 포인트**: 사용자의 거절을 기억하고 점점 다르게 제안하며, 끈질기게 방해하지 않는다.

## 5. ⚠️ Cross-check 발화 보장 체크리스트

라이브에서 불일치 카드가 안 뜨는 사고를 막기 위해, 내부 파일을 아래대로 만든다 (`services/verification/crosscheck/pipeline.py` 동작 기준):

- [ ] 불일치 숫자는 **표가 아니라 본문 prose 문장**에 둔다. (마크다운 표 행 `| ... |`은 문장 추출에서 제외 — 표 수치는 `table_query` 담당)
- [ ] 지표 명칭(영업이익/매출/출산율 등)을 **숫자 바로 앞 24자 이내**에 둔다. (라벨 매칭용)
- [ ] 외부값과 **같은 종류**(둘 다 % 또는 둘 다 절댓값)이고 **단위가 같다**(조원↔조원).
- [ ] 두 숫자의 차이가 **1.2배 이내**여야 한다. (예: 15.8 vs 16.4 = 1.04배 ✅ / 2.5 vs 2.0 = 1.25배 ❌)
- [ ] **같은 연도**를 명시한다. (서로 다른 연도면 의도적으로 비교에서 제외됨)
- [ ] 내부값이 어떤 외부 출처와도 우연히 일치하지 않게 한다. (외부가 내부값을 확증하면 flag 대상에서 빠짐)
- [ ] 단발성 한 자리 정수(1·2·4 등)는 근거로 안 쓰인다 → 의미 있는 다자리/소수/% 값을 쓴다.
- [ ] 형식: 본문 검증용은 `.docx/.pdf/.md/.txt`, 표 질의용은 `.csv/.xlsx`.

> **가장 확실한 방법**: 리허설에서 조사 결과에 실제로 등장하는 외부 수치를 먼저 확인하고, 내부 파일 숫자를 그 값의 ±5~15%로 역산해 맞춘다.
